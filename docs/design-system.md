# Design System — WESON MUSEUM

## テーマ: Luminescent Gallery

> 「高貴な美術館空間 × VR サイバーパンクの光」

物理美術館の静謐さと重厚さを基調に、VR 空間の発光・浮遊感・デジタルノイズをアクセントとして重ねるデザイン。
過剰なサイバーパンクではなく、**美術館の品格が主、テクノロジーの光が副**という関係を守る。

---

## カラーシステム

### ブランドカラー

| 名前 | 値 | 用途 |
|------|-----|------|
| Main | `#34B2AA` | ブランドアイデンティティ、インタラクティブ要素、グロー |
| Accent | `#FCFA0A` | CTA、ハイライト、エネルギーの瞬き |

### 色彩理論的根拠

**`#34B2AA`（テール/シアン寄り）**
- 青（信頼・技術）と緑（生命・創造）の中間に位置する
- 光の波長としてはデジタルディスプレイ・LED に近い印象を与える
- 暗背景上でのコントラスト比が高く、アクセシビリティに有利
- VR ヘッドセットのインターフェース光やホログラムを想起させる

**`#FCFA0A`（エレクトリックイエロー）**
- `#34B2AA` に対して**スプリット補色**に近い関係（緑-黄の方向）
- 高彩度・高輝度で「エネルギーの放出」「電気的な光」を表現
- 美術館の金色（ゴールド）とも読める二重性を持つ
- 少量の使用で最大のインパクトを得られる（アクセントの法則）

### 背景レイヤー構造

```
Layer 0: --color-bg (#050b0a)         ← ページの地 (最も暗い)
Layer 1: --color-bg-1 (#090f0e)       ← カード・パネル
Layer 2: --color-bg-2 (#0e1917)       ← ホバー・強調
Layer G: --color-bg-glass             ← 半透明ガラス面
```

背景にはわずかにテール系の色味を混ぜることで（純黒ではなく `#050b0a`）、
空間の深みとブランドカラーとの統一感を生み出す。

### テキスト階層

| 変数 | 値 | 用途 |
|------|-----|------|
| `--color-text` | `#dff0ee` | 本文・見出し |
| `--color-text-muted` | `#6a9e9a` | 補足情報・ラベル |
| `--color-text-subtle` | `#2e5653` | 装飾・非本質情報 |
| `--color-text-on-main` | `#050b0a` | メイン背景上テキスト |
| `--color-text-on-accent` | `#050b0a` | アクセント背景上テキスト |

### グロー（発光）エフェクト

暗背景上に光源があるという物理的シミュレーション。

```css
--glow-main: 0 0 20px rgba(52,178,170,0.25), 0 0 60px rgba(52,178,170,0.08)
--glow-main-strong: 0 0 30px rgba(52,178,170,0.5), ...
--glow-accent: 0 0 20px rgba(252,250,10,0.2), ...
--glow-accent-strong: 0 0 30px rgba(252,250,10,0.45)
```

グローは**ホバー・アクティブ時のみ強調**し、デフォルトでは抑制する。常時点灯は品格を下げる。

## タイポグラフィ

### フォントファミリー

| 用途 | フォント | 特徴 |
|------|---------|------|
| Display / 見出し | Helvetica Neue → Arial → System | 幾何学的・モダン・美術館のサインを想起 |
| Body / 本文 | + 日本語 (Hiragino, Noto Sans JP) | 読みやすさ重視 |
| Mono / ラベル・データ | JetBrains Mono → Fira Code → Courier | サイバーパンク的データ表示 |

**等幅フォントの戦略的使用**: ナビゲーション、ステータス表示、ナンバリングに等幅フォントを使うことで、「データ的・テクノロジー的」な印象を局所的に与える。見出し・本文には使わない。

### タイプスケール (modular scale 1.25)

```
text-xs:   0.64rem  (10.2px) — モノラベル、タグ
text-sm:   0.80rem  (12.8px) — 補足情報
text-base: 1.00rem  (16.0px) — 本文
text-lg:   1.25rem  (20.0px) — リードコピー
text-xl:   1.56rem  (25.0px) — サブ見出し
text-2xl:  1.95rem  (31.0px) — セクション見出し
text-3xl:  2.44rem  (39.0px) — ページ見出し
text-4xl:  3.05rem  (48.8px) — 大見出し
text-hero: clamp(3rem, 8vw, 6rem) — ヒーロータイトル
```

### レタースペーシング

```
tracking-tight:   -0.02em  — 大きなディスプレイ文字
tracking-wide:     0.08em  — 見出し
tracking-wider:    0.15em  — ナビゲーション
tracking-widest:   0.25em  — 等幅ラベル、タグ
```

**広いレタースペーシングを積極的に使う**: 美術館のサイン・ラベルはゆとりのある字間が品格を生む。

## コンポーネント原則

### Card

```css
/* デフォルト: 落ち着いた存在感 */
background: var(--color-bg-1);
border: 1px solid var(--color-border);  /* 薄いテールボーダー */

/* ホバー: テールのグローで浮かび上がる */
border-color: var(--color-border-bright);
box-shadow: var(--shadow-panel), var(--glow-main);
transform: translateY(-2px);
```

### Button

| バリアント | 用途 |
|-----------|------|
| `btn-primary` | メイン CTA（テール塗り） |
| `btn-outline` | セカンダリ（テール枠） |
| `btn-accent` | 特別強調（黄色塗り、多用禁止） |

### Tag / Badge

等幅フォント + 大文字 + ワイドトラッキング + テールボーダー。
コンテンツの分類・ステータス表示に使用。

### Mono Label

データ的な補助情報（数字、ステータス、カテゴリ）を表示するためのパターン。
必ず等幅フォント + `text-transform: uppercase` + `letter-spacing: widest`。

## アニメーション

### 原則

1. **デフォルトは静止** — 不要なアニメーションはつけない
2. **ホバー時のフィードバック** — インタラクティブ要素は必ず視覚的変化
3. **グローはホバー時のみ強化** — 常時グローは安っぽくなる
4. **サイバーパンクアニメは局所的に** — パルス・ローテーションはアイコン的要素のみ

### トランジション

```css
--transition-fast: 150ms cubic-bezier(0.4, 0, 0.2, 1)  /* ホバー応答 */
--transition-base: 250ms cubic-bezier(0.4, 0, 0.2, 1)  /* 一般的遷移 */
--transition-slow: 400ms cubic-bezier(0.4, 0, 0.2, 1)  /* ページ要素 */
```

## 背景テクスチャ

### グリッドパターン

```css
background-image:
  linear-gradient(rgba(52,178,170,0.015) 1px, transparent 1px),
  linear-gradient(90deg, rgba(52,178,170,0.015) 1px, transparent 1px);
background-size: 80px 80px;
```

極めて薄い（不透明度 1.5%）テールのグリッド線。Blueprint・VR 空間のグリッドを暗示する。
**意識に上らないレベル**が適切な強度。

## サイバーパンク要素の使い方

これらの要素は品格を壊さない範囲で使う。過剰は禁止。

| 要素 | 使い方 | 使ってはいけない場所 |
|------|--------|---------------------|
| グローエフェクト | ホバー時、アクティブ要素 | テキスト本文 |
| 等幅フォント | ラベル、数値、ステータス | 見出し、本文 |
| パルスアニメ | ステータスドット | 大面積のエリア |
| 回転リング | ローディング的表現、装飾グラフィック | ナビゲーション |
| スキャンライングリッド | 背景の極薄テクスチャ | フォアグラウンドコンテンツ |
| `//` コメント記法 | セクション区切りラベル | 頻繁すぎる使用 |

## アクセシビリティ

- **コントラスト比**: `#34B2AA` on `#050b0a` ≈ 8.5:1（WCAG AAA 達成）
- **`#FCFA0A`** on `#050b0a` ≈ 16:1（最高水準）
- **フォーカスリング**: アクセントカラー（黄）で視認性を確保
- **モーション**: `prefers-reduced-motion` 対応（将来対応予定）
- **代替テキスト**: 装飾的な SVG・アニメには `aria-hidden="true"`

## 将来的な拡張

### Three.js 導入時のデザイン方針

- 3D シーンの背景色は `--color-bg`（#050b0a）に合わせる
- Three.js の光源色はメインカラー（テール）とアクセント（黄）から選ぶ
- UI オーバーレイには glassmorphism パターン（`.glass` クラス）を使用
- 3D ビューアの外側には通常の HTML コンテンツを配置し、SEO を維持

### ダークモード

現状はダークモード専用。将来的にライトモードが必要になった場合は、
CSS カスタムプロパティを `@media (prefers-color-scheme: light)` で上書きする形で対応できる。
